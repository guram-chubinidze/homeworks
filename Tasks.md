## # 1. პაროლის გენერატორი და ვალიდატორი
* **ინპუტი:**
    * პაროლის სიგრძე
    * სიმბოლოები
    * რიცხვები
    * დიდი/პატარა ასოები
* **ვალიდატორი:** მხოლოდ ლათინური სიმბოლოები.
* **ლოგიკა / ფუნქციონალი:**
    1.  **ფუნქცია 1:** თუ პარამეტრი `True`-ა, პაროლის მთლიან სიგრძეს ყოფს ინპუტში მოსული აქტიური პარამეტრების რაოდენობის მიხედვით.
    2.  **ფუნქცია 2:** მოსული პარამეტრების მიხედვით აგენერირებს შესაბამის სტრინგებს (სიმბოლოების ბაზას).
    3.  **ფუნქცია 3:** რაოდენობების მიხედვით ჭრის ამ სტრინგებს (მაგ. პირველი 2, 4, 6 სიმბოლო და ა.შ.).
* *შენიშვნა:* აქ შეიძლება პაროლის შეფასების ფუნქციის გამოძახებაც (იხ. დავალება #2).

## # 2. პაროლის სიმტკიცის შეფასების სისტემა
* **პაროლის პარამეტრები და ქულები:**
    * **სიგრძე:** მინიმუმ 0, მაქსიმუმ -2 ქულა (`6` სიმბოლო = 0 ქულა; `8` სიმბოლო = 1 ქულა; `10 >` სიმბოლო = 2 ქულა).
    * **ციფრები:** 2 ქულა
    * **სიმბოლოები:** 2 ქულა
    * **დიდი ასოები:** 2 ქულა
    * **პატარა ასოები:** 2 ქულა
    * **გამეორება (ჯარიმები):** ციფრების ან ანბანის (დიდი და პატარა გაერთიანებული) მიმდევრობითი გამეორებისას: `3` ერთნაირი = -1 ქულა, `4` ერთნაირი = -2 ქულა.
* **მთავარი დამატებითი ფუნქცია:** * ეს ფუნქცია უნდა ამოწმებდეს არა მხოლოდ 1 სიმბოლოს დამთხვევას, არამედ გამეორებებზე: `2` გამეორება = -1 ქულა, `3 >=` გამეორება = -2 ქულა.
* **საბოლოო შეფასება:**
    * `0-დან 7-მდე ქულა` -> **Weak** (სუსტი)
    * `8-დან 9-მდე ქულა` -> **Medium** (საშუალო)
    * `10 ქულა` -> **Strong** (ძლიერი)

## # 3. ფიბონაჩის მიმდევრობის კალკულატორი
* **ინპუტი:** რიცხვი
* **ვალიდატორი:** ამოწმებს რიცხვებს და ციფრებს, აბრუნებს შესაბამის შეტყობინებებს (ციფრის ვალიდაციაც).
* **ლოგიკა / ფუნქციონალი:** ფიბონაჩის ფუნქცია, რომელიც ჩერდება მაშინ, როდესაც მიმდევრობის სიგრძე შემოტანილ რიცხვს მიაღწევს ან გასცდება.
* **აქ მთავარი გამომრჩა შესაბამისი პასუხი დავუბრუნო ვალიდაციაზე

## # 4. ტექსტის დამუშავება და პალინდრომი
* **ინპუტი:** ტექსტი
* **ვალიდატორი:**  მხოლოდ ასოებსა და ციფრებს იღებს.
* **ლოგიკა / ფუნქციონალი:**
    1.  სფეისებისა და სასვენი ნიშნების მოშორება, ორმაგი სფეისების (Double spaces) გასუფთავება.
    2.  **ფუნქცია:** ყოფს ტექსტს 2 ნაწილად, აბრუნებს (reverse) მეორე ნაწილს და ამოწმებს, რამდენი სიმბოლო არ ემთხვევა ერთმანეთს.
    3.  თუ მხოლოდ `1` სიმბოლო არ ემთხვევა, მომხმარებელს სთავაზობს სიმბოლოს დამატებას/წაშლას (ან ზოგად რედაქტირებას).

## # 5. ზედმეტსახელების გენერატორი
* **ინპუტი:** სიტყვა
* **ვალიდატორი:** მხოლოდ ლათინური სიმბოლოები, სიტყვების რაოდენობის შემოწმება.
* **ლოგიკა / ფუნქციონალი:**
    * სფეისებისა და სასვენი ნიშნების მოშორება.
    * **ზედმეტსახელის 5 სავარაუდო შაბლონი (გენერირდება 7 ვარიანტიდან):**
        1. სახელი + `_` + მიმდინარე წელი
        2. სახელი + `-` + მიმდინარე წელი
        3. სახელი + შემთხვევითი წელი (2000-2026)
        4. სახელი + მიმდინარე წელი
        5. სახელი + ერთნაირი რიცხვები (რიცხვების რაოდენობა შემთხვევით 2-დან 4-მდე)
        6. სახელი + `-` + ერთნაირი რიცხვები (რაოდენობა 2-4)
        7. სახელი + `_` + ერთნაირი რიცხვები (რაოდენობა 2-4)
    * ამ შაბლონებიდან პროგრამამ უნდა აარჩიოს **5 შემთხვევითი ვარიანტი** და გამოიტანოს.
    * უკვე გამოტანილი ვარიანტები უნდა შეინახოს, რათა მომხმარებლის მიერ ხელახლა მოთხოვნისას (თუ არ მოეწონა) იგივე ვარიანტები არ განმეორდეს.

## # 6. რიცხვების სორტირება
* **ინპუტი:** სფეისით გამოყოფილი რიცხვები და სორტირების ტიპის არგუმენტი:
    * `d` - დაღმავალი (Descending)
    * `a` - აღმავალი (Ascending)
    * `r` - შემთხვევითი (Random)
    * `u` - უნიკალური (Unique)
* **ვალიდატორი:** მხოლოდ რიცხვები და სფეისები.
* **ლოგიკა / ფუნქციონალი:**
    1.  ორმაგი სფეისების გასუფთავება.
    2.  **ფუნქცია:** გადაცემული არგუმენტის (`d`, `a`, `r`, `u`) მიხედვით აბრუნებს შესაბამისად დასორტირებულ სიას.

## # 7. ტექსტის წმენდა
* **ინპუტი:** ტექსტი

## # 8. რიცხვების სიის შემცირება და ჯამი
* **ინპუტი:** მძიმეებით გამოყოფილი რიცხვების სია.
* **ვალიდატორი:** ნებას რთავს მხოლოდ რიცხვებსა და მძიმეებს.
* **ლოგიკა / ფუნქციონალი:**
    1.  ტექსტის სფეისებისგან გასუფთავება.
    2.  **ფუნქცია:** სტრინგის/სიის პირველ ორ ინდექსზე მდგომ ელემენტს გარდაქმნის `int`-ად, კრებს მათ და მიღებულ ჯამს ინახავს სიის (list) ბოლოში. ეს პროცესი მეორდება მანამ, სანამ სიის/სტრინგის სიგრძე `1`-ზე მეტია (სანამ 1 ელემენტი არ დარჩება).
    3.  ინპუტში შემოსული საწყისი მონაცემებიც უნდა შეინახოს პირველ ელემენტად.
    4.  ბოლოში იბეჭდება მიღებული ლისტის მთლიანი მონაცემები.

## # 9. დუბლიკატი სიტყვების მთვლელი
* **ინპუტი:** ტექსტი
* **ლოგიკა / ფუნქციონალი:**
    1.  ტექსტის გასუფთავება სფეისებისგან და სასვენი ნიშნებისგან, შემდეგ კი მისი დაყოფა სიტყვების ლისტად (`.split()`).
    2.  გამოიყენება `count` მეთოდი სიტყვების სიხშირის დასათვლელად.
    3.  ფუნქციამ უნდა დააბრუნოს ის სიტყვები, რომლებიც მეორდება `2 ან მეტჯერ`. ეს სიტყვები ინახება ახალ ლისტში და სორტირდება მათი გამეორების რაოდენობის მიხედვით.

## 🚀 Git & Branching სტრატეგია
პროექტის ვერსიების კონტროლისთვის გამოიყენება შემდეგი განტოტების (Branch) სისტემა:

1.  **`main` ბრენჩი:** * აქ სრულდება და იფუშება (push) **1, 2, 3** ამოცანები.
2.  **`secondbranch` ბრენჩი:** * აქ სრულდება **4, 5, 6** ამოცანები და კეთდება Pull Request (PR) `main`-ში.
3.  **`thirdbranch` ბრენჩი:** * აქ სრულდება **7, 8, 9, 10** ამოცანები და კეთდება Pull Request (PR) `main`-ში.

📌 *დანართი:* Git ბრძანებების სია დამატებულია ვიზუალური სურათის სახით.
